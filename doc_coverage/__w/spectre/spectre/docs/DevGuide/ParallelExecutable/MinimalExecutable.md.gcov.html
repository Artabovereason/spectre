       5             : # The minimal SpECTRE executable {#tutorial_minimal_parallel_executable}
       6             : 
       7             : This tutorial will illustrate what is needed to compile the minimal
       8             : SpECTRE executable that will simply print some useful information
       9             : about the executable and then exit.
      10             : 
      11             : Specifically, this tutorial will introduce:
      12             : - `add_spectre_parallel_executable`, a CMake function that will add a
      13             :   build target for a SpECTRE executable
      14             : - A user-provided `Metavariables`, a C++ struct that is used to
      15             :   specify the metaprogram that is converted into a C++ executable.
      16             : - how to build a SpECTRE executable
      17             : - how to run a SpECTRE executable
      18             : - useful information that can be extracted from all SpECTRE executables
      19             : - `Main`, the main parallel component that acts as the main function
      20             :   of a C++ executable.
      21             : 
      22             : In this tutorial, `SPECTRE_ROOT` refers to the directory in which
      23             : SpECTRE was cloned, and `SPECTRE_BUILD_DIR` refers to the directory in
      24             : which you built SpECTRE.
      25             : 
      26             : ### Creating a build target for a parallel executable
      27             : 
      28             : The first step is to select (or create) a directory (within
      29             : `SPECTRE_ROOT/src`) that will hold the files from which the executable
      30             : will be created.  For this tutorial we created the directory
      31             : `ParallelTutorial` in `src/Executables/Examples`.  If you create a new
      32             : directory for the executable, you will need to edit the
      33             : `CMakeLists.txt` file in its parent directory and add (replacing
      34             : `ParallelTutorial` with the appropriate directory name)
      35             : 
      36             : \snippet Examples/CMakeLists.txt add_subdirectory
      37             : 
      38             : The second step is to edit (or create) the `CMakeLists.txt` file in
      39             : the executable directory and add a call to
      40             : `add_spectre_parallel_executable` such as:
      41             : 
      42             : \snippet ParallelTutorial/CMakeLists.txt add_spectre_executable
      43             : 
      44             : The (SpECTRE defined) `CMake` function `add_spectre_executable` takes
      45             : five arguments:
      46             : 
      47             : - The name of the executable (in the given example `MinimalExample`)
      48             :   that will be created, which will also be the name of the
      49             :   corresponding build target.
      50             : - The base name (in the given example `MinimalExecutable`) of the two
      51             :   user-provided files that declare (`MinimalExecutableFwd.hpp`) and
      52             :   define (`MinimalExecutable.hpp`) a C++ struct (or struct template)
      53             :   that will describe the metaprogram that is converted into a C++
      54             :   executable.  We will refer to this struct (template) as the
      55             :   metavariables struct (template), and the files declaring and
      56             :   defining the metavariables struct (template) as the metavariables
      57             :   files.
      58             : - The path relative to `SPECTRE_ROOT/src` to the directory holding the
      59             :   metavariables files (in the given example
      60             :   `Executables/Examples/ParallelTutorial`)
      61             : - The specific metavariables struct (in the given example
      62             :   `Metavariables`) that will be used to create the executable.  This
      63             :   is either the name of the metavariables struct in the metavariables
      64             :   files, or a specific instantiation of the metavariables struct
      65             :   template in the metavariables files.
      66             : - A list of SpECTRE libraries that the executable will be linked
      67             :   against (in the given example the `CMake` list `LIBS_TO_LINK` where
      68             :   `%Informer` and `Utilities` are two SpECTRE libraries)
      69             : 
      70             : ### Writing the metavariables files
      71             : 
      72             : The header files from which an executable is generated must declare
      73             : and define a metavariables struct that can be thought of as a
      74             : compile-time input file that defines what the executable will do.
      75             : 
      76             : The first metavariables file (`MinimalExecutableFwd.hpp`) is simply a
      77             : forward declaration of the metavariables struct:
      78             : 
      79             : \snippet MinimalExecutableFwd.hpp metavariables_forward_declaration
      80             : 
      81             : (Note that `#%pragma once` tells the compiler to only include the file
      82             : once per compilation, and the `/// \%cond` and `/// \%endcond` around
      83             : the code tells doxygen not to generate documentation from the wrapped
      84             : region)
      85             : 
      86             : The second metavariables file (`MinimalExecutable.hpp`) holds the
      87             : definition of the metavariables struct.
      88             : 
      89             : \snippet MinimalExecutable.hpp metavariables_definition
      90             : 
      91             : The metavariables struct must define an enum class `Phase` with the
      92             : phases of the executable (which must include `Initialization` and
      93             : `Exit`), and a static function `determine_next_phase`.  In this
      94             : example, no additional phases are defined, and the executable will
      95             : execute the `Initialization` phase followed by the `Exit` phase.
      96             : (`Parallel::CProxy_ConstGlobalCache` is an unused proxy to the
      97             : `ConstGlobalCache` that is explained below)
      98             : 
      99             : The metavariables struct must define a type alias `component_list`
     100             : that is a `tmpl::list` (a typelist defined in `Utilities/TMPL.hpp`) of
     101             : the parallel components used by the executable.  In this example no
     102             : parallel components are used.
     103             : 
     104             : The metavariables struct must define `help`, a `static constexpr
     105             : OptionString` that will be printed as part of the help message of the
     106             : executable. (`OptionString` is defined in `Options/Options.hpp`.)
     107             : 
     108             : In addition to defining the metavaribles struct, the metavariables
     109             : file must define the two vectors of functions `charm_init_node_funcs`
     110             : and `charm_init_proc_funcs`
     111             : 
     112             : \snippet  MinimalExecutable.hpp executable_example_charm_init
     113             : 
     114             : that are executed at startup by Charm++ on each node and processing
     115             : element (PE) the executable runs on.  In this example, the vectors are
     116             : empty.
     117             : 
     118             : ### Building a SpECTRE executable
     119             : 
     120             : Let `$EXECUTABLE` be the name of the executable (the first argument
     121             : passed to the `add_spectre_executable` `CMake` function, so in this
     122             : example `MinimalExample`).  Then the executable can be built by
     123             : running the following command in `$SPECTRE_BUILD_DIR`:
     124             : 
     125             : ```
     126             : make $EXECUTABLE
     127             : ```
     128             : 
     129             : which will produce the executable of the same name in
     130             : `$SPECTRE_BUILD_DIR/bin`.
     131             : 
     132             : ### Running a SpECTRE executable
     133             : 
     134             : To run a SpECTRE executable, run the command:
     135             : 
     136             : ```
     137             : ./$SPECTRE_BUILD_DIR/bin/$EXECUTABLE  <options>
     138             : ```
     139             : 
     140             : where `<options>` must include any required command-line options.  In
     141             : the simple example for this tutorial, no command-line options are

     192             : ### Extracting useful information from a SpECTRE executable
     193             : 
     194             : Every SpECTRE executable comes with a set of command-line options that
     195             : can be used to obtain useful information about the executable (and for
     196             : executables expecting an input file, whether or not the input file can
     197             : be parsed successfully).
     198             : 
     199             : #### Getting a list of available options
     200             : 
     201             : To get a list of available options for a SpECTRE executable, run
     202             : either of the following commands:
     203             : 
     204             : ```
     205             : ./$SPECTRE_BUILD_DIR/bin/$EXECUTABLE  --help
     206             : ./$SPECTRE_BUILD_DIR/bin/$EXECUTABLE  -h
     207             : ```
     208             : 
     209             : In the middle of the `Charm++` startup information will now appear a
     210             : long list of `Charm++` related command-line options which we will
     211             : ignore for this tutorial.  After the SpECTRE startup information,
     212             : there is now a list of available command-line options:
     213             : 
     214             : ```
     215             :   -h [ --help ]             Describe program options
     216             :   --check-options           Check input file options
     217             :   --dump-source-tree-as arg If specified, then a gzip archive of the source
     218             :                             tree is dumped with the specified name. The archive
     219             :                             can be expanded using 'tar -xzf ARCHIVE.tar.gz'
     220             :   --dump-paths              Dump the PATH, CPATH, LD_LIBRARY_PATH,
     221             :                             LIBRARY_PATH, and CMAKE_PREFIX_PATH at compile
     222             :                             time.
     223             :   --dump-environment        Dump the result of printenv at compile time.
     224             :   --dump-library-versions   Dump the contents of SpECTRE's LibraryVersions.txt
     225             :   --dump-only               Exit after dumping requested information.
     226             : ```
     227             : 
     228             : which we will describe in detail below. This is followed by a
     229             : description of the expected input-file options, which in this example
     230             : will simply print the `help` string from the metavariables struct, and
     231             : the statement that there are no options expected:
     232             : 
     233             : ```
     234             : ==== Description of expected options:
     235             : A minimal executable
     236             : 
     237             : <No options>
     238             : ```
     239             : 
     240             : We will cover input file options in a future tutorial.
     241             : 
     242             : #### Checking options
     243             : 
     244             : As the minimal executable in this tutorial expects no input-file
     245             : options, running the command:
     246             : 
     247             : ```
     248             : ./$SPECTRE_BUILD_DIR/bin/$EXECUTABLE  --check-options
     249             : ```
     250             : 
     251             : will print out `No options to check!` and exit the program.  See a
     252             : future tutorial for an example of checking input-file options.
     253             : 
     254             : #### Dumping the source tree
     255             : 
     256             : In order to aid in reproducibility, all SpECTRE executables contain a
     257             : copy of `$SPECTRE_ROOT`.  To obtain the source tree as an archive
     258             : file, run the command:
     259             : 
     260             : ```
     261             : ./$SPECTRE_BUILD_DIR/bin/$EXECUTABLE  --dump-source-tree-as SpECTRE
     262             : ```
     263             : 
     264             : which will produce the archive file `SpECTRE.tar.gz` which can be
     265             : expanded with the command:
     266             : 
     267             : ```
     268             : tar -xzf SpECTRE.tar.gz
     269             : ```
     270             : 
     271             : #### Dumping other information about a SpECTRE executable
     272             : 
     273             : In addition to dumping the entire source tree there are the following options:
     274             : 
     275             : - `--dump-paths` will print out various paths from when the
     276             :   executable was compiled
     277             : - `--dump-environment` will print the results of `printenv`
     278             :   (i.e. all of the environment variables) from when the executable was
     279             :   compiled
     280             : - `--dump-library-versions` will print the contents of SpECTRE's
     281             :   `LibraryVersions.txt` which will contain the versions of libraries
     282             :   that SpECTRE linked against, as well as various `CMake` variables
     283             : 
     284             : Also note that the option `--dump-only` can be used to have the
     285             : SpECTRE executable terminate immediately after dumping the
     286             : information.
     287             : 
     288             : ### Behind the scenes:  the main parallel component
     289             : 
     290             : Every Charm++ executable needs a mainchare.  All SpECTRE executables
     291             : use `Main` (found in `src/Parallel/Main.hpp`) as the mainchare.  When
     292             : a SpECTRE executable is run, the constructor of `Main` that is executed
     293             : takes the command line options as an argument (as type
     294             : `CkArgMsg*`).  This constructor performs the following operations:
     295             : 
     296             : - Prints useful startup information
     297             : - Parses the command line options, performing any requested operations
     298             : - Parses the input file options, populating a tagged tuple with their
     299             :   values.  (This is discussed in more detail in a future tutorial on
     300             :   input file options.)
     301             : - Creates the `ConstGlobalCache` (a nodegroup chare) that holds
     302             :   objects created from input file options that are constant and stored
     303             :   once per Charm++ node, as well as proxies to all other parallel
     304             :   components.
     305             : - Creates user-requested non-array parallel components, passing them
     306             :   the proxy to the `ConstGlobalCache` as well as any items they
     307             :   request that can be created from input file options.
     308             : - Creates empty array user-requested parallel components
     309             : - Sends the complete list of parallel components to the
     310             :   `ConstGlobalCache`.
     311             : 
     312             : Once the list of parallel components is sent to the `ConstGlobalCache`
     313             : on each node, the `Main` member function
     314             : `allocate_array_components_and_execute_initial_phase` will be
     315             : executed.  This will allocate the elements of the array parallel
     316             : components by calling the `allocate_array` member function of each
     317             : component.  Then the `start_phase` member function is called on each
     318             : component, which will execute the phase action list for the
     319             : `Initialization` phase for each component.  Charm++ will execute each
     320             : phase until it detects that nothing is happening (quiescence
     321             : detection).  As this represents a global synchronization point, the
     322             : number of phases should be minimized in order to exploit the power of
     323             : SpECTRE.  After each phase, the `execute_next_phase` member function
     324             : of `Main` will be called.  This member function first determines what
     325             : the next phase is.  If the next phase is `Exit`, then some useful
     326             : information is printed and the program exits gracefully.  Otherwise the
     327             : `execute_next_phase` member function of each parallel component is
     328             : called.
